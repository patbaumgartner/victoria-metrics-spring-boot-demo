# Code of Conduct

This project is intended to be a practical, public example of a Spring Boot and
VictoriaMetrics observability stack. To keep it useful for everyone, please be
respectful and constructive.

## Expected Behavior

- Be courteous and professional.
- Focus on the technical issue, not the person.
- Assume good intent and ask clarifying questions when needed.
- Give clear, specific feedback that helps improve the project.
- Respect maintainers' time and decisions.

## Unacceptable Behavior

- Harassment, discrimination, or demeaning language.
- Personal attacks, trolling, or deliberate disruption.
- Posting private or sensitive information without permission.
- Repeatedly ignoring maintainer guidance or project boundaries.

## Enforcement

Maintainers may remove or edit content, close issues or pull requests, and
restrict participation when behavior is harmful to the project or its
contributors.

## Reporting Problems

If you experience or witness behavior that violates this code of conduct,
please report it privately using [SECURITY.md](SECURITY.md) or by opening a
minimal issue that asks to be contacted privately.

## Scope

This code of conduct applies to all project spaces, including GitHub issues,
pull requests, discussion threads, and any project-managed communication
channels.
